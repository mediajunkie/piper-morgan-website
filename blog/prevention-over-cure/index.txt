Expensive prevention now creates cheap operations forever. It was Tuesday. My regular workday was done, and I had time to turn to Piper Morgan before dinner.</p><h3>The symptom appears</h3><p>Foundation branch merge complete. 186 files changed. +47,677 insertions, -1,334 deletions. Developers assume it’s fine (imports work!). But tests fail mysteriously. Type checkers get confused. Strict tooling breaks.</p><p>You don’t know there’s a problem for 137 days. Then suddenly: ImportError.</p><p><strong>Analysis complete</strong>: 20,000+ word root cause document. Timeline reconstructed. Problem understood. Fixes identified.</p><p>Not just “fix the imports.” But “understand why this happened and prevent it systematically.”</p><h3>The cure: Quick fixes (what we didn’t do)</h3><p>Let’s talk about what the quick-fix approach might look like:</p><h4>Quick Fix Option 1: Manual file creation (10 minutes)</h4><pre># Create the missing __init__.py files<br>touch services/api/__init__.py<br>touch services/integrations/mcp/__init__.py<br># ... But conservatively 10:1. Optimistically 100:1. Realistically somewhere between.</p><h3>Why prevention feels expensive</h3><p>Here’s why teams don’t invest in prevention:</p><h4>Problem 1: Immediate pain vs delayed benefit</h4><p>Quick fix: Pain stops NOW (dopamine hit)<br>Prevention: Pain stops LATER (no immediate reward)</p><p>Human brains prefer immediate reward (LLM “brains” too… they got it from us!). Can’t spend 3.5 hours on infrastructure!”</p><p>But mathematics: 3.5 hours invested = 10+ hours saved over project lifetime = net +6.5 hours.</p><p>False economy: Saving time now costs more time later.</p><p>Fortunately, this is a side project without deadlines but for most efforts this is a real issue.</p><h4>Problem 4: Unclear when “enough” prevention</h4><p>“Should we build 1 hook or 5? prevented</strong>: Every time someone reads the docs instead of asking (10+ times over feature lifetime)<br><strong class="markup--strong markup--p-strong">Time per prevention</strong>: 15–30 minutes<br><strong class="markup--strong markup--p-strong">ROI</strong>: 5:1 to 20:1 over feature lifetime</p><p name="212e" id="212e" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Pattern</strong>: Prevention consistently provides 5–20x ROI over time</p><h3 name="cc31" id="cc31" class="graf graf--h3 graf-after--p">What makes for effective prevention?</h3><p name="ff41" id="ff41" class="graf graf--p graf-after--h3">Not all prevention is equal. Effective prevention has characteristics:</p><h4 name="7485" id="7485" class="graf graf--h4 graf-after--p">Automated enforcement</h4><p name="7c91" id="7c91" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Good</strong>: Pre-commit hook prevents bad commits (zero human attention)<br><strong class="markup--strong markup--p-strong">Bad</strong>: Documentation says “remember to do X” (requires human remembering)</p><p name="500e" id="500e" class="graf graf--p graf-after--p">Humans forget. Automation doesn’t. Best prevention requires no human intervention.</p><h4 name="811f" id="811f" class="graf graf--h4 graf-after--p">Comprehensive scope</h4><p name="f70a" id="f70a" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Good</strong>: Prevents entire problem class (“enforce Python package structure”)<br><strong class="markup--strong markup--p-strong">Bad</strong>: Prevents specific symptom (“add this one missing file”)</p><p name="c818" id="c818" class="graf graf--p graf-after--p">Narrow fixes prevent one symptom. Broad fixes prevent all symptoms in problem class.</p><h4 name="e5f7" id="e5f7" class="graf graf--h4 graf-after--p">Clear documentation</h4><p name="3828" id="3828" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Good</strong>: 60+ lines explaining requirement, rationale, verification<br><strong class="markup--strong markup--p-strong">Bad</strong>: Code comment saying “needed for tests”</p><p name="400f" id="400f" class="graf graf--p graf-after--p">Future developers need to understand WHY prevention exists. Otherwise they bypass or remove it.</p><h4 name="a3e7" id="a3e7" class="graf graf--h4 graf-after--p">Easy verification</h4><p name="9248" id="9248" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Good</strong>: <code class="markup--code markup--p-code">python -c &quot;import services.api&quot;</code> tests if structure is correct<br><strong class="markup--strong markup--p-strong">Bad</strong>: &quot;Run full test suite and see if it works&quot;</p><p name="dd9f" id="dd9f" class="graf graf--p graf-after--p">Quick verification enables fast feedback. Slow verification gets skipped.</p><h4 name="df91" id="df91" class="graf graf--h4 graf-after--p">Appropriate severity</h4><p name="51d1" id="51d1" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Good</strong>: Missing <code class="markup--code markup--p-code">__init__.py</code> blocks commit (critical requirement)<br><strong class="markup--strong markup--p-strong">Good</strong>: Misnamed manual test warns (useful but not critical)<br><strong class="markup--strong markup--p-strong">Bad</strong>: Everything blocks (too strict, gets bypassed)<br><strong class="markup--strong markup--p-strong">Bad</strong>: Everything warns (too lenient, gets ignored)</p><p name="29fb" id="29fb" class="graf graf--p graf-after--p">Match enforcement to severity. Critical requirements block. Useful guidance warns.</p><h3 name="e61d" id="e61d" class="graf graf--h3 graf-after--p">The investment decision framework</h3><p name="1a7b" id="1a7b" class="graf graf--p graf-after--h3">How do you decide when to invest in prevention?</p><h4 name="9ab9" id="9ab9" class="graf graf--h4 graf-after--p">Question 1: How often will this recur?</h4><p name="3102" id="3102" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Once ever</strong>: Quick fix appropriate<br><strong class="markup--strong markup--p-strong">Occasionally (1–2 times/year)</strong>: Prevention probably worth it<br> <strong class="markup--strong markup--p-strong">Frequently (monthly)</strong>: Prevention definitely worth it<br><strong class="markup--strong markup--p-strong">Every commit</strong>: Prevention mandatory</p><h4 name="10ef" id="10ef" class="graf graf--h4 graf-after--p">Question 2: How expensive is each occurrence?</h4><p name="d3fa" id="d3fa" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">5 minutes</strong>: Prevention needs to be very cheap to justify<br><strong class="markup--strong markup--p-strong">30 minutes</strong>: Prevention justified if problem recurs 3+ times<br><strong class="markup--strong markup--p-strong">2+ hours</strong>: Prevention justified if problem recurs even once<br><strong class="markup--strong markup--p-strong">Unknown/variable</strong>: Prevention provides insurance value</p><h4 name="766c" id="766c" class="graf graf--h4 graf-after--p">Question 3: Can it be automated?</h4><p name="1b08" id="1b08" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Yes</strong>: Strong case for prevention (minimal ongoing cost)<br><strong class="markup--strong markup--p-strong">Partially</strong>: Moderate case (reduced future cost)<br><strong class="markup--strong markup--p-strong">No</strong>: Weaker case (still requires manual work)</p><h4 name="dbe6" id="dbe6" class="graf graf--h4 graf-after--p">Question 4: How certain is the solution?</h4><p name="e3ac" id="e3ac" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Very certain</strong>: Invest in comprehensive prevention<br><strong class="markup--strong markup--p-strong">Somewhat certain</strong>: Invest in lightweight prevention, iterate<br><strong class="markup--strong markup--p-strong">Uncertain</strong>: Quick fix, observe, invest in prevention after second occurrence</p><h4 name="d5e8" id="d5e8" class="graf graf--h4 graf-after--p">Question 5: What’s the blast radius?</h4><p name="84f4" id="84f4" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Affects one developer</strong>: Lower priority for prevention<br><strong class="markup--strong markup--p-strong">Affects team</strong>: Medium priority<br><strong class="markup--strong markup--p-strong">Affects production</strong>: High priority<br><strong class="markup--strong markup--p-strong">Affects users</strong>: Mandatory prevention</p><p name="f05a" id="f05a" class="graf graf--p graf-after--p"><strong

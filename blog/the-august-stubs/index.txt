The ConversationRepository stubs weren’t a new bug — they were a seven-month-old assumption that “the plumbing works” finally getting tested.</p><p name="8832" id="8832" class="graf graf--p graf-after--p">It did feel good to be on the move again, and by now this cycle of discovering and completing unfinished work is almost too familiar.</p><h3 name="5b87" id="5b87" class="graf graf--h3 graf-after--p">The 75% pattern strikes again</h3><p name="ed8a" id="ed8a" class="graf graf--p graf-after--h3">This was familiar. Too familiar.</p><p name="7afa" id="7afa" class="graf graf--p graf-after--p">We’d documented Pattern-045 (“Green Tests, Red User”) weeks earlier: tests pass because they mock interfaces, but the actual implementation is incomplete. The standup templates. The project repository. Now the conversation repository.</p><p name="2b51" id="2b51" class="graf graf--p graf-after--p">The pattern keeps recurring because it’s structural, not accidental:</p><p name="d38b" id="d38b" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Phase 1</strong>: Build the interface. Define the contract. Write tests against mocks.</p><p name="969e" id="969e" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Phase 2</strong>: Implement <em class="markup--em markup--p-em">something</em>. Enough that it doesn’t crash. Log statements to show activity.</p><p name="112b" id="112b" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Phase 3</strong>: Move on. The tests pass. The interface exists. It <em class="markup--em markup--p-em">looks</em> complete.</p><p name="8c21" id="8c21" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Phase 4</strong>: Much later, discover the implementation was never finished.</p><p name="254f" id="254f" class="graf graf--p graf-after--p">We’d been calling it the “75% completion trap.” The August stubs made it concrete. Infrastructure can exist for months — database tables, repository classes, method signatures — while the actual functionality remains a no-op.</p><h3 name="ae0d" id="ae0d" class="graf graf--h3 graf-after--p">What enabled the velocity</h3><p name="b031" id="b031" class="graf graf--p graf-after--h3">Twenty-three issues in three days isn’t just about volume. It comes down to planning and process.</p><p name="2399" id="2399" class="graf graf--p graf-after--p">The gameplans were thorough. Not just “here’s what to build” but “here’s exactly how to verify it’s done.” Phase 0 through Phase N, each with explicit completion criteria.</p><p name="7970" id="7970" class="graf graf--p graf-after--p">The completion matrices worked. Every issue had checkboxes that required evidence. Not “I think it’s done” but “here’s the test output, here’s the commit, here’s the verification.”</p><p name="9e84" id="9e84" class="graf graf--p graf-after--p">The template compliance paid off. The Lead Developer had audited issues against templates, applied fixes for compliance. That rigor during planning meant fewer surprises during implementation.</p><p name="8b7e" id="8b7e" class="graf graf--p graf-after--p">The agents knew their roles. Subagents could be deployed with clear prompts. Code review followed patterns. The methodology had become machinery.</p><h3 name="77f7" id="77f7" class="graf graf--h3 graf-after--p">The fix was fast</h3><p name="a47e" id="a47e" class="graf graf--p graf-after--h3">Once found, the ConversationRepository took an hour to fix. Real SQLAlchemy queries. Actual database writes. The kind of implementation that should have existed from the start.</p><p name="e9a5" id="e9a5" class="graf graf--p graf-after--p">The fix being fast was the point. The <em class="markup--em markup--p-em">finding</em> was the hard part. Seven months of assuming the plumbing worked. Seven months of tests that passed because they never touched the real code.</p><p name="5097" id="5097" class="graf graf--p graf-after--p">The methodology caught it not because we went looking for old bugs, but because moving fast through the codebase means touching everything. Velocity is a diagnostic tool. When you can’t move fast, something is blocking you. When you move fast and things break, you’ve found the gaps.</p><h3 name="6969" id="6969" class="graf graf--h3 graf-after--p">What the stubs taught me (this time)</h3><p name="2439" id="2439" class="graf graf--p graf-after--h3">Infrastructure is not implementation. Tables existing doesn’t mean the queries work. Interfaces defined doesn’t mean the methods do anything. Tests passing doesn’t mean the feature is complete.</p><p name="7c5b" id="7c5b" class="graf graf--p graf-after--p">The completion matrices we’d been enforcing — the explicit evidence requirements, the verification steps, the STOP conditions — exist precisely because the alternative is seven months of empty methods.</p><p name="cf59" id="cf59" class="graf graf--p graf-after--p graf--trailing">I don’t regret discovering these August stubs. They taught us something.

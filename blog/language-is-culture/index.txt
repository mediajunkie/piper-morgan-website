1b:T6329,<figure><img alt="In office cubicles a robot mention “Cathedral doctrine” while a person mentioned “Inchworm protocol”" src="https://cdn-images-1.medium.com/max/1024/1*8bk2ww0_8D6xyAL8TT0ceQ.png" /><figcaption>“Don’t forget the Excellence Flywheel!”</figcaption></figure><p><em>October 24 to November 6</em></p><p>“August broke the semantic analyzer.” This sentence appears in November 4’s session log. It’s simultaneously technical problem description, humor about meta-analysis, and cultural artifact. Will anyone reading Piper Morgan documentation six months from now will encounter this phrase and understand? “We analyze our own development patterns. Sometimes our analysis tools fail hilariously. We laugh about it while fixing it.”</p><p>That’s language creating culture.</p><p>“The 75% pattern.” “Phase −1 investigation.” “Cathedral Doctrine.” “Time Lord Philosophy.” “The Excellence Flywheel.” “Archaeological discovery.” “Completion matrix.”</p><p>These aren’t standard software engineering terms. They’re our terms. They emerged through methodological documentation. They get reinforced through repeated use. They create shared understanding between human and AI agents. They encode lessons learned. They transmit methodology.</p><p>This is about how language shapes methodology. How documentation creates culture. How curated logs transmit values. And how this happens on AI teams similarly — but differently — than human teams.</p><h3>The linguistic archaeology</h3><p>Let’s trace some key phrases through their evolution:</p><h3>“Phase −1” (Emerged October 2025)</h3><p><strong>First appearance</strong>: Gameplan template v6.0, “Phase -1: Discovery &amp; Assumption Verification”</p><p><strong>Context</strong>: After multiple instances of starting implementation only to discover existing infrastructure, we formalized investigation as mandatory first phase.</p><p><strong>Evolution</strong>:</p><ul><li>Initially: “Check what exists before building”</li><li>Then: “Discovery phase”</li><li>Finally: “Phase -1 infrastructure verification”</li></ul><p><strong>Why this name</strong>: Standard project phases start at 0 or 1. By calling investigation “Phase −1,” we signal it happens <em>before</em> normal work begins. It’s pre-implementation requirement, not optional preliminary.</p><p><strong>Cultural meaning</strong>: “Phase −1” now carries weight. When agent says “beginning Phase −1 investigation,” everyone knows: We’re checking assumptions before acting. We’re preventing unnecessary work. We’re applying archaeological approach.</p><p>The phrase encodes methodology. Anyone who reads “Phase −1” in documentation understands: Investigation before implementation. Always.</p><h3>“The 75% Pattern” (Emerged November 2025)</h3><p><strong>First appearance</strong>: November 1 discovery of DocumentService</p><p><strong>Context</strong>: Saturday sprint found DocumentService 75% complete. Sunday found todo infrastructure 75% complete. Monday found ActionHumanizer infrastructure complete. Pattern recognized.</p><p><strong>Evolution</strong>:</p><ul><li>Saturday: “DocumentService exists but not wired”</li><li>Sunday: “Todo infrastructure discovered 75% complete”</li><li>Monday: “Another 75% pattern”</li><li>Tuesday-forward: “The 75% Pattern” (capitalized, recognized phenomenon)</li></ul><p><strong>Why this name</strong>: Specific percentage (75%) makes it concrete. Not “mostly done” or “almost finished” but quantified state. Database/service/domain layers (75%) exist, integration layers (25%) missing.</p><p><strong>Cultural meaning</strong>: “75% pattern” signals: Don’t assume missing. Investigate. Infrastructure probably exists at lower layers. Also encodes understanding: Good architecture builds deep (database/services) before wide (integration/UI).</p><p><strong>Linguistic precision</strong>: Could have called it “incomplete infrastructure pattern” or “unfinished features.” But “75% pattern” captures specific nature: Not 50% (half-done). Not 90% (nearly complete). But 75% (foundation solid, wiring missing).</p><h3>“Cathedral Doctrine” (Emerged August 2025)</h3><p><strong>First appearance</strong>: ADR-000, “Meta-Platform Architecture”</p><p><strong>Context</strong>: Need to communicate long-term architectural vision to AI agents who live in present moment. How do you get agents to build toward 3-year vision when they only see current task?</p><p><strong>Evolution</strong>:</p><ul><li>Initially: “Long-term architecture vision”</li><li>Then: “Cathedral metaphor” (building cathedrals took generations)</li><li>Finally: “Cathedral Doctrine” (formal methodological principle)</li></ul><p><strong>Why this name</strong>: Cathedrals were built across generations. Workers who laid foundations never saw completion. Required shared vision transmitted through documentation. Perfect metaphor for AI-human collaboration where “generations” are different Claude instances.</p><p><strong>Cultural meaning</strong>: When architect invokes “Cathedral Doctrine,” it means: This decision serves 3-year vision, not immediate need. Build for future capabilities. Document vision so future agents understand intent.</p><h3>“Time Lord Philosophy” (Emerged September 2025)</h3><p><strong>First appearance</strong>: Session management protocols</p><p><strong>Context</strong>: Development work often spans multiple days. Traditional project management uses hours/days. But Claude sessions don’t map to human time. Need different temporal framework.</p><p><strong>Evolution</strong>:</p><ul><li>Initially: “Don’t work through the night”</li><li>Then: “Session-based time management”</li><li>Finally: “Time Lord Philosophy” (playful, memorable)</li></ul><p><strong>Why this name</strong>: Doctor Who reference. Time Lords experience time non-linearly. Similarly, AI agents don’t experience continuous time. Each session is fresh context. Playful name makes serious methodological point memorable. It also emphasizes that the PM dictates facts to the LLMs. If I say we have 30 hours today, then so be it.</p><p><strong>Cultural meaning</strong>: “Time Lord Philosophy” signals: Time is fluid in AI-human collaboration. Sessions can span days. Work can pause and resume. Don’t force artificial urgency. Use playful time units (jiffy, fortnight) to emphasize non-standard temporal relationship.</p><p><strong>The humor</strong>: Calling it “Time Lord Philosophy” rather than “Asynchronous Session Management” makes it more likely to be remembered and applied. Humor aids transmission.</p><h3>“Completion Matrix” (Emerged November 2025)</h3><p><strong>First appearance</strong>: November 1, Saturday sprint preventing 80% pattern twice</p><p><strong>Context</strong>: Code works but documentation missing. Or documentation exists but evidence not collected. Or evidence collected but related docs not updated. Three levels of completion all required.</p><p><strong>Evolution</strong>:</p><ul><li>Initially: “Make sure it’s really done”</li><li>Then: “Three-level completion: code, docs, process”</li><li>Finally: “Completion Matrix” (formalized framework)</li></ul><p><strong>Why this name</strong>: Matrix implies intersection of dimensions. Code level × documentation level × process level = complete when ALL cells satisfied.</p><p><strong>Cultural meaning</strong>: “Completion Matrix” signals: Done means truly done. Not just code works. Not just documented. But all three levels verified. Prevents 80% pattern (stopping when “feels done” without systematic verification).</p><h3>How language transmits methodology</h3><p>The phrases aren’t just labels. They’re methodology transmission mechanisms.</p><h4>Example 1: Phase −1 becomes automatic</h4><p><strong>Before formalization</strong>: Agent starts implementation. Discovers existing infrastructure mid-work. Wastes time.</p><p><strong>After “Phase </strong>−<strong>1” named</strong>: Agent sees gameplan template. Reads “Phase −1: Infrastructure Verification.” Understands: Investigate first, always.</p><p><strong>Transmission</strong>: The phrase “Phase −1” in gameplan templates teaches methodology without explicit instruction. New agent reads template, sees Phase −1, investigates before implementing. Methodology transmitted through language.</p><h4>Example 2: 75% Pattern enables recognition</h4><p><strong>Before naming</strong>: Each discovery feels unique. “Oh, DocumentService exists.” “Oh, todos built.” No pattern recognized.</p><p><strong>After “75% Pattern” named</strong>: Third discovery: “Another 75% pattern.” Fourth discovery: “Standard 75% pattern.” Pattern becomes visible through naming.</p><p><strong>Transmission</strong>: Naming the pattern helps recognize it. When agent investigates and finds infrastructure, they document: “75% pattern confirmed.” Future agents reading logs see pattern, investigate more thoroughly themselves.</p><h4>Example 3: Cathedral Doctrine aligns decisions</h4><p><strong>Before formalization</strong>: Agent makes good short-term decision that blocks long-term architecture. PM has to explain why decision is wrong. Frustrating.</p><p><strong>After “Cathedral Doctrine” named</strong>: Agent reads ADR-000. Sees “Cathedral Doctrine: Build for 3-year vision.” Makes decision aligned with long-term architecture. Less human correction needed.</p><p><strong>Transmission</strong>: The doctrine name + documentation creates shared understanding. Agent doesn’t need PM to explain each time why long-term matters. The principle is encoded in language.</p><h3>The linguistic feedback loop</h3><p>Language doesn’t just transmit methodology. It refines methodology through usage.</p><h4>The loop:</h4><p><strong>1. Problem encountered</strong>: Investigation reveals pattern (archaeological discovery)</p><p><strong>2. Language emerges</strong>: Name the pattern (“75% pattern”)</p><p><strong>3. Documentation captures</strong>: Logs record instances, explain pattern</p><p><strong>4. Agents adopt</strong>: Future agents read logs, use terminology</p><p><strong>5. Usage refines</strong>: Agents apply pattern, discover edge cases</p><p><strong>6. Language evolves</strong>: Terminology adjusts to capture refinements</p><p><strong>7. Loop continues</strong>: Better language → better understanding → better application → better language</p><h4>Example: “Phase −1” evolution</h4><p><strong>Original</strong>: “Check if infrastructure exists before building”</p><p><strong>Problem</strong>: Too vague. What infrastructure? How to check? When to stop?</p><p><strong>Refinement 1</strong>: “Phase −1: Discovery phase”</p><p><strong>Problem</strong>: “Discovery” too general. Discovery of what?</p><p><strong>Refinement 2</strong>: “Phase −1: Infrastructure Verification”</p><p><strong>Better</strong>: More specific. But what counts as verification?</p><p><strong>Refinement 3</strong>: “Phase −1: Infrastructure Verification (15–30 minutes minimum, all layers, symbol search)”</p><p><strong>Current state</strong>: Sustained duration, explicit methodology, tool specification.</p><p>The language evolved through usage. Each agent applying it encountered ambiguity. Refinements addressed ambiguity. Now “Phase -1” has precise meaning transmitted through documentation.</p><h3>Human teams vs AI teams: Similar but different</h3><p>Language creates culture on human teams too. But different dynamics:</p><h4>Human team language evolution:</h4><p><strong>Emergence</strong>: Informal. Phrase appears in standup, catches on, spreads through conversation.</p><p><strong>Transmission</strong>: Social. New team member overhears phrase, asks meaning, gets explanation.</p><p><strong>Refinement</strong>: Organic. Team discusses, phrase evolves through dialogue.</p><p><strong>Persistence</strong>: Cultural memory. Senior members remember origin stories.</p><p><strong>Failure mode</strong>: Tribal knowledge. New members miss context, use phrases incorrectly.</p><h4>AI team language evolution:</h4><p><strong>Emergence</strong>: Formal. Phrase appears in documentation, gets adopted through reading. Individual human provides 90% of the novely and unique personality. (Though I will admit the LLMs named our “Excellence Flywheel.”)</p><p><strong>Transmission</strong>: Documentation. New agent reads logs/templates, applies terminology.</p><p><strong>Refinement</strong>: Explicit. Ambiguity surfaces, documentation updated, meaning clarified.</p><p><strong>Persistence</strong>: Written record. Every agent can trace phrase origin through git history.</p><p><strong>Failure mode</strong>: Documentation lag. Verbal understanding advances faster than written docs.</p><h4>The hybrid (human + AI collaboration):</h4><p><strong>Emergence</strong>: Human recognizes pattern, AI helps formalize naming.</p><p><strong>Transmission</strong>: Both. Humans explain verbally, AI agents read documentation.</p><p><strong>Refinement</strong>: Collaborative. Humans identify ambiguity, AI agents propose clarifications.</p><p><strong>Persistence</strong>: Dual memory. Human cultural knowledge + comprehensive written record.</p><p><strong>Advantage</strong>: Best of both. Informal emergence with formal documentation. Organic refinement with explicit capture.</p><h3>The glossary we need</h3><p>It usually takes me longer than it should to realize a product needs a glossary. That has happened again with Piper Morgan, where I am fnally realizing this, siz months in. It has become especially pressing now that for the first time I have other human people looking at the project and advising me on the architecture and code.</p><p>Our jargon has grown organically. Time to document systematically. The list below is an initial scan that my comms chief put together when preparing a draft of this article for me, but in the near future I will do another systematic sweep of project docs to tease out any other terms of art we’ve developed, and then I am going to edit the definitions to make sure no meaning has drifted or “flattened.”</p><h4>Core Methodology Terms (relatively unique to this project)</h4><p><strong>Phase </strong>−<strong>1</strong>: Infrastructure verification before implementation. 15–30 minutes minimum. Investigates all layers (database, services, routes, handlers). Uses symbol search tools. Outputs: infrastructure map + gap analysis. Mandatory before Phase 0.</p><p><strong>75% Pattern</strong>: Infrastructure 75% complete at lower layers (database/services/domain) but missing 25% integration layers (routes/handlers/UI). Result of good architecture built “deep before wide.” Requires wiring not rebuilding.</p><p><strong>Cathedral Doctrine</strong>: Build for 3-year vision, not immediate need. Document long-term architecture so future agents understand intent. Named for medieval cathedrals built across generations.</p><p><strong>Time Lord Philosophy</strong>: Time is fluid in AI-human collaboration. Sessions span days. Work pauses and resumes. No artificial urgency. Playful time units (jiffy, fortnight) emphasize non-standard temporal relationship.</p><p><strong>Completion Matrix</strong>: Three-level completion requirement. Level 1: Code (tests passing, integration working). Level 2: Documentation (summaries, evidence, updates). Level 3: Process (criteria met, related docs updated). All levels required for “done.”</p><p><strong>Archaeological Approach</strong>: Systematic investigation to find existing infrastructure before building new. Consistent 18–27:1 ROI. Also called “Phase −1 investigation.”</p><p><strong>Excellence Flywheel</strong>: Small improvements compound. Each success (and failure) teaches a lesson. Each lesson improves next success. Positive feedback loop of methodology refinement.</p><p><strong>Inchworm Protocol</strong>: Systematic completion before proceeding. Finish phase completely, verify thoroughly, document comprehensively, then advance. Prevents 80% pattern.</p><p><strong>Verification Pyramid</strong>: Three tiers of validation. Bottom: Unit tests (fast, many). Middle: Integration tests (slower, fewer). Top: E2E tests (slowest, focused). All three tiers required.</p><p><strong>Validation-Driven Development</strong>: Tests aren’t afterthought. They’re confidence mechanism. Comprehensive tests enable bold changes. 66 tests passing → migration is safe.</p><h4>Cultural Artifacts (arise from conversation)</h4><p><strong>“August broke the semantic analyzer”</strong>: Meta-learning moment. Analysis tools revealing own limits. Going meta on meta. Humor about methodology complexity.</p><p><strong>“Different rhythms, different stages”</strong>: Development velocity varies appropriately. June 100% → October 15%, both healthy. Match work to stage.</p><p><strong>“The long winding road”</strong>: Non-linear path from start to done. Estimates revise as understanding deepens. Discovery-driven development, not failure.</p><p><strong>“Two parallel realities”</strong>: Database sophisticated, web basic. Layers exist in different architectural universes. Gap invisible from within either reality.</p><p><strong>“Accidentally enterprise-ready”</strong>: Infrastructure created for one purpose, suitable for another. Good practices accumulating into unexpected capability.</p><h4>Process Terms (relatively ordinary)</h4><p><strong>Gameplan</strong>: Comprehensive implementation plan. Phases defined, acceptance criteria clear, stop conditions explicit. Template v9.0+ includes Phase -1 mandatory.</p><p><strong>Omnibus Log</strong>: Synthesis of multiple session logs into comprehensive narrative. Pattern-020 format. Creates complete story from fragmented sessions.</p><p><strong>Session Log</strong>: Single agent/session documentation. Chronicles decisions, discoveries, outcomes. Raw material for omnibus logs.</p><p><strong>ADR (Architectural Decision Record)</strong>: Documents significant architecture decisions. Problem, options considered, decision made, rationale. Numbered sequentially (ADR-001, ADR-002…).</p><p><strong>STOP Conditions</strong>: Explicit criteria for stopping work. Prevents scope creep. Examples: “Stop when tests pass” or “Stop when 80% coverage reached.”</p><p><strong>Acceptance Criteria</strong>: Specific, verifiable completion requirements. “All tests passing” or “Documentation updated” or “Related routes verified.”</p><p><strong>Evidence</strong>: Concrete proof of completion. Commit SHAs, test results, SQL logs, performance metrics. Not opinion, measurable.</p><h3>What a glossary might enable</h3><p>Formalizing terminology may feel bureaucratic but ideally it should be a scaling mechanism.</p><h4>Before glossary:</h4><p>Agent: “Should I check if infrastructure exists?”</p><p>PM: “Yes, do Phase −1 investigation. Check all layers. Use Serena for symbol search. Spend 15–30 minutes…”</p><p><strong>Cost</strong>: 5 minutes explanation per agent, every time</p><h3>With glossary:</h3><p>Agent: “Should I check if infrastructure exists?”</p><p>PM: “Yes, do Phase −1”</p><p>Agent: <em>[reads glossary entry if unfamiliar with term, understands complete methodology]</em></p><p><strong>Cost</strong>: 10 seconds, perfect transmission</p><h3>The multiplication:</h3><ul><li>5 agents × 10 investigations each = 50 explanations</li><li>Without glossary: 50 × 5 minutes = 250 minutes (4.2 hours)</li><li>With glossary: 50 × 10 seconds = 8.3 minutes</li><li><strong>Savings</strong>: 4 hours of explanation time</li></ul><p>But really that’s the kind of math AI’s like to do. The more important impact is consistency. A glossary ensures every agent gets same explanation. No drift. No telephone-game degradation. Perfect transmission every time.</p><h3>Language as culture transmission</h3><p>Here’s the key insight: <strong>Methodology without language is intuition. Methodology with language is culture.</strong></p><p>I can intuitively understand that investigating before building saves time. But until we named it “Phase −1” and documented it systematically, each agent had to learn from scratch.</p><p>Now: Agent reads “Phase −1” in template. Looks up glossary. Understands ROI. Sees four discovery examples. Applies methodology. No explanation needed.</p><p><strong>Language encoded methodology. Documentation transmitted culture.</strong></p><p>This is different from human teams where senior members carry cultural knowledge in memory. With AI agents:</p><ul><li>No cultural memory between sessions</li><li>No informal transmission through conversation</li><li>No senior members modeling behavior</li><li><strong>All transmission through documentation</strong></li></ul><p>Which means: <strong>Language precision matters more</strong>. On human teams, vague phrases get clarified through context and discussion. With AI agents, vague phrases stay vague unless documentation clarifies.</p><h3>Today’s meta-observation</h3><p>Writing this post <em>is</em> literally an example of the pattern it describes.</p><p>By documenting how language creates culture, we’re using language to transmit cultural understanding about how language transmits culture. Meta-level.</p><p>Future agents reading this will understand:</p><ul><li>Phrases encode methodology</li><li>Documentation transmits culture</li><li>Glossaries enable scaling</li><li>Language precision matters</li><li>This understanding itself transmitted through language</li></ul><p>The recursion is intentional. Methodology about methodology requires meta-language. “August broke the semantic analyzer” is funny because we’re aware we’re going meta. This post is deliberately self-referential because the subject is language creating self-awareness.</p><h3>What this means for other AI-human teams</h3><p>The pattern applies beyond Piper Morgan:</p><p><strong>Any team collaborating with AI agents should</strong>:</p><ol><li><strong>Name patterns explicitly</strong>: Don’t leave methodology implicit. When you recognize pattern, name it. Make it concrete through language.</li><li><strong>Document systematically</strong>: Don’t rely on verbal transmission. AI agents need written records. Comprehensive documentation enables perfect transmission.</li><li><strong>Create glossaries</strong>: Formalize terminology as jargon emerges. Glossary enables scaling, ensures consistency, reduces explanation overhead.</li><li><strong>Refine through usage</strong>: Language isn’t static. As agents apply methodology, capture edge cases, update documentation, improve precision.</li><li><strong>Embrace humor</strong>: Playful phrases (“Time Lord Philosophy,” “August broke…”) are more memorable than dry technical terms. Humor aids transmission.</li><li><strong>Go meta intentionally</strong>: Occasionally step back and analyze the language itself. How is methodology transmitted? What’s working? What needs clarification?</li></ol><p>The hybrid human-AI team has advantage: Human cultural intuition + AI documentation discipline. Best of both worlds if you leverage it intentionally.</p><h3>The living glossary</h3><p>Glossary shouldn’t be static document. It should evolve as methodology evolves:</p><p><strong>When new pattern emerges</strong>: Add entry. Name it, define it, document instances.</p><p><strong>When terminology drifts</strong>: Update entry. Clarify meaning, add examples, distinguish from similar terms.</p><p><strong>When usage reveals ambiguity</strong>: Refine entry. Add precision, specify edge cases, include anti-patterns.</p><p><strong>When better name appears</strong>: Replace entry. Document evolution, explain why new term is clearer.</p><p>The glossary is living document reflecting evolving culture. Not dictionary of fixed meanings. But changelog of methodology maturation.</p><p>And maintaining it is cultural practice: When agent encounters vague terminology, they propose clarification. When human explains concept repeatedly, they formalize in glossary. When methodology evolves, documentation updates.</p><p><strong>Language creates culture. Documentation maintains culture. Refinement improves culture.</strong></p><p><em>Next on Building Piper Morgan we resume the build narrative on November 24 with “From 15 Agents to One User.”</em></p><p><em>Do your teams develop shared jargon? Agent doesn’t need PM to explain each time why long-term matters. The principle is encoded in language.</p><h3 name="2f33" id="2f33" class="graf graf--h3 graf-after--p">The linguistic feedback loop</h3><p name="c074" id="c074" class="graf graf--p graf-after--h3">Language doesn’t just transmit methodology. It refines methodology through usage.</p><h4 name="6356" id="6356" class="graf graf--h4 graf-after--p">The loop:</h4><ol class="postList"><li name="a735" id="a735" class="graf graf--li graf-after--h4"><strong class="markup--strong markup--li-strong">Problem encountered</strong>: Investigation reveals pattern (archaeological discovery)</li><li name="db5f" id="db5f" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Language emerges</strong>: Name the pattern (“75% pattern”)</li><li name="b797" id="b797" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Documentation captures</strong>: Logs record instances, explain pattern</li><li name="907f" id="907f" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Agents adopt</strong>: Future agents read logs, use terminology</li><li name="5325" id="5325" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Usage refines</strong>: Agents apply pattern, discover edge cases</li><li name="3493" id="3493" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Language evolves</strong>: Terminology adjusts to capture refinements</li><li name="f93e" id="f93e" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Loop continues</strong>: Better language → better understanding → better application → better language</li></ol><h4 name="bfb9" id="bfb9" class="graf graf--h4 graf-after--li">Example: “Phase −1” evolution</h4><p name="5c9d" id="5c9d" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Original</strong>: “Check if infrastructure exists before building”</p><p name="4f1a" id="4f1a" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Problem</strong>: Too vague. What infrastructure? How to check? When to stop?</p><p name="9685" id="9685" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Refinement 1</strong>: “Phase −1: Discovery phase”</p><p name="dc57" id="dc57" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Problem</strong>: “Discovery” too general. Discovery of what?</p><p name="417e" id="417e" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Refinement 2</strong>: “Phase −1: Infrastructure Verification”</p><p name="6175" id="6175" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Better</strong>: More specific. But what counts as verification?</p><p name="8858" id="8858" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Refinement 3</strong>: “Phase −1: Infrastructure Verification (15–30 minutes minimum, all layers, symbol search)”</p><p name="b180" id="b180" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Current state</strong>: Sustained duration, explicit methodology, tool specification.</p><p name="d990" id="d990" class="graf graf--p graf-after--p">The language evolved through usage. Each agent applying it encountered ambiguity. Refinements addressed ambiguity. Now “Phase -1” has precise meaning transmitted through documentation.</p><h3 name="5530" id="5530" class="graf graf--h3 graf-after--p">Human teams vs AI teams: Similar but different</h3><p name="3e57" id="3e57" class="graf graf--p graf-after--h3">Language creates culture on human teams too. But different dynamics:</p><h4 name="76ba" id="76ba" class="graf graf--h4 graf-after--p">Human team language evolution:</h4><p name="267c" id="267c" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Emergence</strong>: Informal. Phrase appears in standup, catches on, spreads through conversation.</p><p name="98e1" id="98e1" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Transmission</strong>: Social. New team member overhears phrase, asks meaning, gets explanation.</p><p name="4b86" id="4b86" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Refinement</strong>: Organic. Team discusses, phrase evolves through dialogue.</p><p name="1572" id="1572" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Persistence</strong>: Cultural memory. Senior members remember origin stories.</p><p name="0a2c" id="0a2c" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Failure mode</strong>: Tribal knowledge. New members miss context, use phrases incorrectly.</p><h4 name="a7ba" id="a7ba" class="graf graf--h4 graf-after--p">AI team language evolution:</h4><p name="62e2" id="62e2" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Emergence</strong>: Formal. Phrase appears in documentation, gets adopted through reading. Individual human provides 90% of the novely and unique personality. (Though I will admit the LLMs named our “Excellence Flywheel.”)</p><p name="ab45" id="ab45" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Transmission</strong>: Documentation. New agent reads logs/templates, applies terminology.</p><p name="b71f" id="b71f" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Refinement</strong>: Explicit. Ambiguity surfaces, documentation updated, meaning clarified.</p><p name="43da" id="43da" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Persistence</strong>: Written record. Every agent can trace phrase origin through git history.</p><p name="62c0" id="62c0" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Failure mode</strong>: Documentation lag. Verbal understanding advances faster than written docs.</p><h4 name="a74f" id="a74f" class="graf graf--h4 graf-after--p">The hybrid (human + AI collaboration):</h4><p name="319a" id="319a" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Emergence</strong>: Human recognizes pattern, AI helps formalize naming.</p><p name="a506" id="a506" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Transmission</strong>: Both. Humans explain verbally, AI agents read documentation.</p><p name="7b08" id="7b08" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Refinement</strong>: Collaborative. Humans identify ambiguity, AI agents propose clarifications.</p><p name="3783" id="3783" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Persistence</strong>: Dual memory. Human cultural knowledge + comprehensive written record.</p><p name="79d2" id="79d2" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Advantage</strong>: Best of both. Informal emergence with formal documentation. Organic refinement with explicit capture.</p><h3 name="7569" id="7569" class="graf graf--h3 graf-after--p">The glossary we need</h3><p name="ea0a" id="ea0a" class="graf graf--p graf-after--h3">It usually takes me longer than it should to realize a product needs a glossary. That has happened again with Piper Morgan, where I am fnally realizing this, siz months in. It has become especially pressing now that for the first time I have other human people looking at the project and advising me on the architecture and code.</p><p name="9787" id="9787" class="graf graf--p graf-after--p">Our jargon has grown organically. Time to document systematically. The list below is an initial scan that my comms chief put together when preparing a draft of this article for me, but in the near future I will do another systematic sweep of project docs to tease out any other terms of art we’ve developed, and then I am going to edit the definitions to make sure no meaning has drifted or “flattened.”</p><h4 name="441c" id="441c" class="graf graf--h4 graf-after--p">Core Methodology Terms (relatively unique to this project)</h4><p name="379d" id="379d" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Phase </strong>−<strong class="markup--strong markup--p-strong">1</strong>: Infrastructure verification before implementation. 15–30 minutes minimum. Investigates all layers (database, services, routes, handlers). Uses symbol search tools. Outputs: infrastructure map + gap analysis. Mandatory before Phase 0.</p><p name="4cc6" id="4cc6" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">75% Pattern</strong>: Infrastructure 75% complete at lower layers (database/services/domain) but missing 25% integration layers (routes/handlers/UI). Result of good architecture built “deep before wide.” Requires wiring not rebuilding.</p><p name="8b33" id="8b33" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Cathedral Doctrine</strong>: Build for 3-year vision, not immediate need. Document long-term architecture so future agents understand intent. Named for medieval cathedrals built across generations.</p><p name="e22d" id="e22d" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Time Lord Philosophy</strong>: Time is fluid in AI-human collaboration. Sessions span days. Work pauses and resumes. No artificial urgency. Playful time units (jiffy, fortnight) emphasize non-standard temporal relationship.</p><p name="37cf" id="37cf" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Completion Matrix</strong>: Three-level completion requirement. Level 1: Code (tests passing, integration working). Level 2: Documentation (summaries, evidence, updates). Level 3: Process (criteria met, related docs updated). All levels required for “done.”</p><p name="321c" id="321c" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Archaeological Approach</strong>: Systematic investigation to find existing infrastructure before building new. Consistent 18–27:1 ROI. Also called “Phase −1 investigation.”</p><p name="fd6a" id="fd6a" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Excellence Flywheel</strong>: Small improvements compound. Each success (and failure) teaches a lesson. Each lesson improves next success. Positive feedback loop of methodology refinement.</p><p name="2f2b" id="2f2b" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Inchworm Protocol</strong>: Systematic completion before proceeding. Finish phase completely, verify thoroughly, document comprehensively, then advance. Prevents 80% pattern.</p><p name="487c" id="487c" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Verification Pyramid</strong>: Three tiers of validation. Bottom: Unit tests (fast, many). Middle: Integration tests (slower, fewer). Top: E2E tests (slowest, focused). All three tiers required.</p><p name="3808" id="3808" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Validation-Driven Development</strong>: Tests aren’t afterthought. They’re confidence mechanism. Comprehensive tests enable bold changes. 66 tests passing → migration is safe.</p><h4 name="dd00" id="dd00" class="graf graf--h4 graf-after--p">Cultural Artifacts (arise from conversation)</h4><p name="a1ed" id="a1ed" class="graf graf--p graf--startsWithDoubleQuote graf-after--h4"><strong class="markup--strong markup--p-strong">“August broke the semantic analyzer”</strong>: Meta-learning moment. Analysis tools revealing own limits. Going meta on meta. Humor about methodology complexity.</p><p name="2018" id="2018" class="graf graf--p graf--startsWithDoubleQuote graf-after--p"><strong class="markup--strong markup--p-strong">“Different rhythms, different stages”</strong>: Development velocity varies appropriately. June 100% → October 15%, both healthy. Match work to stage.</p><p name="cd30" id="cd30" class="graf graf--p graf--startsWithDoubleQuote graf-after--p"><strong class="markup--strong markup--p-strong">“The long winding road”</strong>: Non-linear path from start to done. Estimates revise as understanding deepens. Discovery-driven development, not failure.</p><p name="2197" id="2197" class="graf graf--p graf--startsWithDoubleQuote graf-after--p"><strong class="markup--strong markup--p-strong">“Two parallel realities”</strong>: Database sophisticated, web basic. Layers exist in different architectural universes. Gap invisible from within either reality.</p><p name="0b16" id="0b16" class="graf graf--p graf--startsWithDoubleQuote graf-after--p"><strong class="markup--strong markup--p-strong">“Accidentally enterprise-ready”</strong>: Infrastructure created for one purpose, suitable for another. Good practices accumulating into unexpected capability.</p><h4 name="f0ae" id="f0ae" class="graf graf--h4 graf-after--p">Process Terms (relatively ordinary)</h4><p name="bf9c" id="bf9c" class="graf graf--p graf-after--h4"><strong class="markup--strong markup--p-strong">Gameplan</strong>: Comprehensive implementation plan. Phases defined, acceptance criteria clear, stop conditions explicit. Template v9.0+ includes Phase -1 mandatory.</p><p name="184c" id="184c" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Omnibus Log</strong>: Synthesis of multiple session logs into comprehensive narrative. Pattern-020 format. Creates complete story from fragmented sessions.</p><p name="2c34" id="2c34" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Session Log</strong>: Single agent/session documentation. Chronicles decisions, discoveries, outcomes. Raw material for omnibus logs.</p><p name="c556" id="c556" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">ADR (Architectural Decision Record)</strong>: Documents significant architecture decisions. Problem, options considered, decision made, rationale. Numbered sequentially (ADR-001, ADR-002…).</p><p name="d946" id="d946" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">STOP Conditions</strong>: Explicit criteria for stopping work. Prevents scope creep. Examples: “Stop when tests pass” or “Stop when 80% coverage reached.”</p><p name="070a" id="070a" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Acceptance Criteria</strong>: Specific, verifiable completion requirements. “All tests passing” or “Documentation updated” or “Related routes verified.”</p><p name="fa33" id="fa33" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Evidence</strong>: Concrete proof of completion. Commit SHAs, test results, SQL logs, performance metrics. Not opinion, measurable.</p><h3 name="aa6d" id="aa6d" class="graf graf--h3 graf-after--p">What a glossary might enable</h3><p name="703e" id="703e" class="graf graf--p graf-after--h3">Formalizing terminology may feel bureaucratic but ideally it should be a scaling mechanism.</p><h4 name="1604" id="1604" class="graf graf--h4 graf-after--p">Before glossary:</h4><p name="16f6" id="16f6" class="graf graf--p graf-after--h4">Agent: “Should I check if infrastructure exists?”</p><p name="95b6" id="95b6" class="graf graf--p graf-after--p">PM: “Yes, do Phase −1 investigation. Check all layers. Use Serena for symbol search. Spend 15–30 minutes…”</p><p name="8b85" id="8b85" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Cost</strong>: 5 minutes explanation per agent, every time</p><h3 name="d810" id="d810" class="graf graf--h3 graf-after--p">With glossary:</h3><p name="8d2a" id="8d2a" class="graf graf--p graf-after--h3">Agent: “Should I check if infrastructure exists?”</p><p name="e865" id="e865" class="graf graf--p graf-after--p">PM: “Yes, do Phase −1”</p><p name="4a6b" id="4a6b" class="graf graf--p graf-after--p">Agent: <em class="markup--em markup--p-em">[reads glossary entry if unfamiliar with term, understands complete methodology]</em></p><p name="beb1" id="beb1" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Cost</strong>: 10 seconds, perfect transmission</p><h3 name="2d80" id="2d80" class="graf graf--h3 graf-after--p">The multiplication:</h3><ul class="postList"><li name="5514" id="5514" class="graf graf--li graf-after--h3">5 agents × 10 investigations each = 50 explanations</li><li name="f3f8" id="f3f8" class="graf graf--li graf-after--li">Without glossary: 50 × 5 minutes = 250 minutes (4.2 hours)</li><li name="8a45" id="8a45" class="graf graf--li graf-after--li">With glossary: 50 × 10 seconds = 8.3 minutes</li><li name="9cb6" id="9cb6" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Savings</strong>: 4 hours of explanation time</li></ul><p name="b1df" id="b1df" class="graf graf--p graf-after--li">But really that’s the kind of math AI’s like to do. The more important impact is consistency. A glossary ensures every agent gets same explanation. No drift. No telephone-game degradation. Perfect transmission every time.</p><h3 name="7814" id="7814" class="graf graf--h3 graf-after--p">Language as culture transmission</h3><p name="fc45" id="fc45" class="graf graf--p graf-after--h3">Here’s the key insight: <strong class="markup--strong markup--p-strong">Methodology without language is intuition. Methodology with language is culture.</strong></p><p name="2b16" id="2b16" class="graf graf--p graf-after--p">I can intuitively understand that investigating before building saves time. But until we named it “Phase −1” and documented it systematically, each agent had to learn from scratch.</p><p name="bcab" id="bcab" class="graf graf--p graf-after--p">Now: Agent reads “Phase −1” in template. Looks up glossary. Understands ROI. Sees four discovery examples. Applies methodology. No explanation needed.</p><p name="37d6" id="37d6" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Language encoded methodology. Documentation transmitted culture.</strong></p><p name="5276" id="5276" class="graf graf--p graf-after--p">This is different from human teams where senior members carry cultural knowledge in memory. With AI agents:</p><ul class="postList"><li name="6216" id="6216" class="graf graf--li graf-after--p">No cultural memory between sessions</li><li name="3bbb" id="3bbb" class="graf graf--li graf-after--li">No informal transmission through conversation</li><li name="6044" id="6044" class="graf graf--li graf-after--li">No senior members modeling behavior</li><li name="87b9" id="87b9" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">All transmission through documentation</strong></li></ul><p name="d756" id="d756" class="graf graf--p graf-after--li">Which means: <strong class="markup--strong markup--p-strong">Language precision matters more</strong>. On human teams, vague phrases get clarified through context and discussion. With AI agents, vague phrases stay vague unless documentation clarifies.</p><h3 name="e490" id="e490" class="graf graf--h3 graf-after--p">Today’s meta-observation</h3><p name="e956" id="e956" class="graf graf--p graf-after--h3">Writing this post <em class="markup--em markup--p-em">is</em> literally an example of the pattern it describes.</p><p name="18cc" id="18cc" class="graf graf--p graf-after--p">By documenting how language creates culture, we’re using language to transmit cultural understanding about how language transmits culture. Meta-level.</p><p name="f965" id="f965" class="graf graf--p graf-after--p">Future agents reading this will understand:</p><ul class="postList"><li name="75ec" id="75ec" class="graf graf--li graf-after--p">Phrases encode methodology</li><li name="9921" id="9921" class="graf graf--li graf-after--li">Documentation transmits culture</li><li name="8039" id="8039" class="graf graf--li graf-after--li">Glossaries enable scaling</li><li name="9da4" id="9da4" class="graf graf--li graf-after--li">Language precision matters</li><li name="727e" id="727e" class="graf graf--li graf-after--li">This understanding itself transmitted through language</li></ul><p name="8025" id="8025" class="graf graf--p graf-after--li">The recursion is intentional. Methodology about methodology requires meta-language. “August broke the semantic analyzer” is funny because we’re aware we’re going meta. This post is deliberately self-referential because the subject is language creating self-awareness.</p><h3 name="eb8b" id="eb8b" class="graf graf--h3 graf-after--p">What this means for other AI-human teams</h3><p name="710e" id="710e" class="graf graf--p graf-after--h3">The pattern applies beyond Piper Morgan:</p><p name="d0a5" id="d0a5" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Any team collaborating with AI agents should</strong>:</p><ol class="postList"><li name="408b" id="408b" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">Name patterns explicitly</strong>: Don’t leave methodology implicit. When you recognize pattern, name it. Make it concrete through language.</li><li name="27a5" id="27a5" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Document systematically</strong>: Don’t rely on verbal transmission. AI agents need written records. Comprehensive documentation enables perfect transmission.</li><li name="f315" id="f315" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Create glossaries</strong>: Formalize terminology as jargon emerges. Glossary enables scaling, ensures consistency, reduces explanation overhead.</li><li name="434c" id="434c" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Refine through usage</strong>: Language isn’t static. As agents apply methodology, capture edge cases, update documentation, improve precision.</li><li name="2235" id="2235" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Embrace humor</strong>: Playful phrases (“Time Lord Philosophy,” “August broke…”) are more memorable than dry technical terms. Humor aids transmission.</li><li name="2d25" id="2d25" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Go meta intentionally</strong>: Occasionally step back and analyze the language itself. How is methodology transmitted? What’s working? What needs clarification?</li></ol><p name="48cc" id="48cc" class="graf graf--p graf-after--li">The hybrid human-AI team has advantage: Human cultural intuition + AI documentation discipline. Best of both worlds if you leverage it intentionally.</p><h3 name="7fec" id="7fec" class="graf graf--h3 graf-after--p">The living glossary</h3><p name="8749" id="8749" class="graf graf--p graf-after--h3">Glossary shouldn’t be static document. It should evolve as methodology evolves:</p><p name="4c39" id="4c39" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">When new pattern emerges</strong>: Add entry. Name it, define it, document instances.</p><p name="1c77" id="1c77" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">When terminology drifts</strong>: Update entry. Clarify meaning, add examples, distinguish from similar terms.</p><p name="3744" id="3744" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">When usage reveals ambiguity</strong>: Refine entry. Add precision, specify edge cases, include anti-patterns.</p><p name="d70d" id="d70d" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">When better name appears</strong>: Replace entry. Document evolution, explain why new term is clearer.</p><p name="a71b" id="a71b" class="graf graf--p graf-after--p">The glossary is living document reflecting evolving culture. Not dictionary of fixed meanings. But changelog of methodology maturation.</p><p name="a60a" id="a60a" class="graf graf--p graf-after--p">And maintaining it is cultural practice: When agent encounters vague terminology, they propose clarification. When human explains concept repeatedly, they formalize in glossary.
